Arquitectura de Software: Sistema de Becas Estudiantiles
El sistema está diseñado bajo un modelo de Validación en Dos Capas (Cliente y Servidor) para garantizar que la información sea veraz antes de tocar la base de datos.

1. El Frontend: Single Page Application (SPA) Dinámica

Inyección Asíncrona (AJAX/Fetch): El sistema no solo oculta capas, sino que utiliza fetch() para traer el HTML y el Script de cada paso solo cuando se necesitan. Esto aligera la carga inicial y mejora la seguridad, ya que el código de la "Pantalla Final" no existe en el navegador hasta que llegas a ella.

Gestor de Memoria Centralizado (formDataStorage): El sistema utiliza un objeto global de JavaScript que actúa como una "base de datos temporal" en la RAM del navegador. Esto permite que, si un usuario vuelve del Paso 5 al Paso 2, sus datos sigan allí (persistencia de sesión).

Identificadores Únicos por Marca de Tiempo: Para la tabla de familiares, se utiliza Date.now(). Esto genera un ID único (ej: 17087000...) para cada fila, permitiendo que el sistema sepa exactamente qué dato borrar del almacenamiento global sin confundir un familiar con otro.

2. Flujo de Navegación y Lógica de Bifurcación
El sistema no es lineal; es inteligente. El "Director de Orquesta" (JS) evalúa el contexto en cada paso:

El Salto Lógico (Skipping): * Escenario: Si en el Paso 2 (trabaja = false), el código detecta este estado booleano.

Acción: Al presionar "Siguiente", el puntero currentStep se modifica manualmente de 2 a 4, saltándose el Paso 3 (Datos Laborales). Esto optimiza la experiencia de usuario (UX) al no pedir datos irrelevantes.

Validación de Bloqueo (Hard-Stop): En el Paso 3, si el usuario llega allí y confirma que tiene trabajo, el sistema ejecuta un bloqueo total: lanza un alert() y detiene el flujo, ya que tener empleo suele ser una causal de exclusión inmediata para esta beca.

3. Procesamiento en el Backend (PHP)
Una vez que el usuario hace clic en "Confirmar" en el Paso 9, el objeto de memoria se envía al servidor donde PHP toma el control.

Sanitización de "Super-Arreglos": PHP recibe los datos y aplica limpieza para evitar ataques (como inyecciones de código). Los datos de los familiares se reciben como arreglos indexados, lo que permite al servidor procesar 1 o 10 familiares usando el mismo bloque de código.

Validación Espejo (Seguridad): Aunque JS ya validó la edad, PHP vuelve a calcularla. Esto es una medida de seguridad: si un usuario avanzado modifica el JS para "engañar" al navegador y decir que tiene 20 años cuando tiene 50, el servidor lo detectará y rechazará la solicitud.

El Recolector de Errores: En lugar de fallar y detenerse, PHP llena un "carrito de errores".

$errores[] = "El índice académico es insuficiente";

$errores[] = "La edad excede el límite permitido";
Si al final el carrito no está vacío, el servidor devuelve una respuesta estructurada con todos los motivos del rechazo.
